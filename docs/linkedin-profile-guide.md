# LinkedIn 账号搭建与内容撰写指南

> 面向在日求职的技术/AI 方向候选人

---

## 一、账号基础设置

### 头像
- 使用**职业感强**的正面照，纯色或简洁背景
- 面部占画面 60% 以上，表情自然有亲和力
- 分辨率建议 400×400px 以上
- 避免：生活照、旅游照、带滤镜的照片

### 背景图（Banner）
- 尺寸：1584×396px
- 内容建议：技术栈关键词 / 所在城市天际线 / 简洁的个人品牌标语
- 免费工具：Canva 有 LinkedIn Banner 模板

### 自定义 URL
将默认乱码 URL 改为 `linkedin.com/in/yourname`：
> 设置 → 公开个人资料和 URL → 编辑个人 URL

## 二、Headline（标题行）

Headline 是 LinkedIn 算法权重最高的字段，也是别人搜索时最先看到的内容。

### 字数限制
最多 **220 字符**（手机端显示约 120 字符，优先在前半段放关键词）

### 写法公式

```
[职能] | [核心技术栈] | [差异化标签]
```

**示例：**
```
AI Engineer | LLM · Agent Systems · Python | Based in Tokyo
```
```
Backend Engineer | Python · AWS · Docker | Seeking AI-first roles in Japan
```
```
Product Engineer | React · TypeScript · LLM Integration | Japan · English · Chinese
```

### 注意事项
- **不要只写职位名称**（如 "Software Engineer"），太泛，无法被精准搜索
- 用 `|` 或 `·` 分隔，比逗号视觉更清晰
- 在日求职建议注明 "Japan" 或 "Tokyo"，方便 recruiter 筛选
- 可加语言能力：`Trilingual: EN · JP · ZH`

## 三、About（关于我）

About 是你的"电梯演讲"，也是 LinkedIn 中可以讲故事的唯一地方。

### 字数限制
最多 **2600 字符**，展开前只显示约 **300 字符**，开头要抓住眼球。

### 结构模板

```
第一句：一句话定义你是谁 + 擅长什么
（300字以内，展开前可见部分）

第一段：背景 + 核心能力
- 几年经验，做过什么类型的工作
- 技术方向和专长

第二段：具体亮点 / 成就
- 1-2 个有数据支撑的项目或成果
- 体现解决问题的能力，而非只列技术名词

第三段：差异化标签
- 语言能力、跨文化背景、对 AI/行业的理解
- 为什么适合在日本工作

结尾：Call to Action
- 当前状态（Open to work / Actively looking）
- 邀请 recruiter 联系的一句话
```

### 示例

```
AI-focused software engineer based in Tokyo, building LLM-powered products and agent systems.

I have X years of experience in backend development and AI integration, with a strong foundation
in Python, AWS, and distributed systems. My recent work focuses on applying LLM and agent
frameworks to real-world products — from RAG pipelines to autonomous task systems.

One project I'm particularly proud of: [简述项目 + 量化结果，如 "reduced processing time by 60%"].

What makes me different: I'm trilingual (Chinese / English / Japanese), which helps me work
effectively in Japan's bilingual tech environments and collaborate with both local and
international teams.

Currently exploring new opportunities in AI engineering or software product roles in Japan.
Feel free to reach out — always happy to connect.
```

### 常见错误
| 错误 | 建议替代 |
|------|---------|
| 只列技术名词 | 结合实际项目说明如何用这些技术 |
| 用第三人称 | 全程用第一人称，更有亲近感 |
| 没有 Call to Action | 最后一段明确表达求职意向 |
| 纯英文（在日求职） | 可用英文为主，结尾加一行日文更友好 |

## 四、Skills（技能）

### 添加方式
最多添加 **50 个技能**，可按优先级排序。

### 推荐策略
- 前 **3 个技能**重点展示（最先被看到），放最强的核心技能
- 技能要与 Headline 和 About 中提到的关键词**保持一致**
- 请同事/朋友帮你**认可（Endorse）**关键技能，提升可信度

### AI/后端方向推荐技能词

**核心技术**
`Python` · `SQL` · `Docker` · `Kubernetes` · `AWS` · `Git`

**AI/LLM 方向**
`Large Language Models (LLM)` · `Prompt Engineering` · `AI Agents` · `RAG` · `LangChain` · `OpenAI API`

**后端/工程**
`FastAPI` · `PostgreSQL` · `Redis` · `REST APIs` · `Microservices` · `CI/CD`

**软技能（在日求职加分）**
`Japanese` · `Business Japanese` · `Cross-cultural Communication`

## 五、Experience（工作经历）

### 每段经历的写法

```
职位名称 | 公司 | 时间段

[1-2 句概述这个角色的职责和背景]

• [成就 1] — 量化结果优先，如 "Reduced API latency by 40% by..."
• [成就 2] — 用 STAR 结构：情境 → 任务 → 行动 → 结果
• [成就 3] — 体现技术深度或业务影响

技术栈：Python · AWS · PostgreSQL · Docker
```

### 写作原则
- **动词开头**：Built / Designed / Led / Optimized / Reduced / Increased
- **量化结果**：有数字比有文字更有说服力
- **不要照搬 JD**：写你实际做了什么，而不是岗位要求

## 六、Featured（精选）

精选区可以放：
- **个人项目链接**（GitHub、Demo 站）
- **技术博客文章**
- **简历 PDF**
- **演讲或视频**

> 推荐至少放 1 个项目链接，让 recruiter 点击后能直接看到你的成果。

---

## 七、Recommendations（推荐信）

- 请求前同事、经理、合作方撰写推荐
- 至少准备 **2-3 条**，尤其是来自直属上级的
- 撰写时建议给对方提供：你希望强调的 2-3 个点 + 具体合作项目名称

---

## 八、Open to Work 设置

### 两种模式
| 模式 | 可见范围 | 适用场景 |
|------|---------|---------|
| 公开绿框（#OpenToWork） | 所有人可见 | 积极求职，不在意现雇主知道 |
| 仅对 Recruiter 可见 | 只有 LinkedIn Recruiter 可见 | 在职但低调找工作 |

### 设置内容
- 目标职位名称（可填多个）
- 工作地点偏好（Tokyo / Remote / Hybrid）
- 到岗时间（Immediately / 1 month / 3 months）
- 雇佣类型（Full-time / Contract）

---

## 九、活跃度建议

LinkedIn 算法偏好活跃账号，建议：

| 行为 | 频率 | 效果 |
|------|------|------|
| 点赞/评论行业内容 | 每天 5-10 分钟 | 提升曝光，让 recruiter 看到你 |
| 发原创内容 | 每 1-2 周一篇 | 建立专业形象 |
| 关注目标公司官页 | 入职前持续关注 | 了解公司动态 |
| 与 recruiter 建立连接 | 主动发 connection request | 建立人脉管道 |

### 值得发的内容类型
- 技术学习笔记（"今天学到了关于 X 的一件事"）
- 项目复盘（做了什么、遇到什么问题、如何解决）
- 行业观察（AI 新工具、新模型发布的看法）
- 求职感悟（真实分享，有高互动率）

## 十、在日求职专项建议

### 日文内容
- Headline 可以中英日三语：`AIエンジニア | Python · LLM | 東京在住`
- About 结尾加 1-2 句日文，表明日语沟通能力

### 日本市场关键词
Recruiter 在日本常用的搜索词：
`エンジニア` · `バックエンド` · `AI` · `機械学習` · `Python` · `AWS` · `東京` · `外資系`

### Connection 策略
1. 先连接在目标公司的 **中国/台湾裔工程师**（同文化背景，更容易回复）
2. 连接 **Tech Recruiter**（标题含 "Recruiter" / "Talent" / "採用"）
3. 连接 **Dev Advocate / Developer Relations**（通常乐于帮助社区）

---

## 快速检查清单

在发送求职申请前，确认以下项目：

- [ ] 头像专业，背景图已设置
- [ ] Headline 含核心技术关键词 + 地点
- [ ] About 前 300 字有吸引力，含 Call to Action
- [ ] 至少 5 段工作/项目经历，有量化成果
- [ ] Skills 填满 15+ 个，前 3 个是最强技能
- [ ] Featured 有至少 1 个项目或简历链接
- [ ] Open to Work 已开启，职位和地点填写准确
- [ ] 自定义 URL 已设置（linkedin.com/in/yourname）
- [ ] 有至少 2 条 Recommendations
